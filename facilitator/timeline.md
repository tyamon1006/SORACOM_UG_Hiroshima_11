# 詳細タイムライン（2時間）

## ⏰ 全体スケジュール

| 時間 | フェーズ | 司会者の行動 | 参加者の活動 |
|------|----------|--------------|--------------|
| 0:00-0:05 | 開始・説明 | 開会挨拶・概要説明 | 着席・資料確認 |
| 0:05-0:30 | GPS機器セットアップ | 手順ガイド・個別サポート | 機器設定作業 |
| 0:30-0:45 | 動作確認 | 確認手順の説明・問題対応 | 動作テスト実施 |
| 0:45-1:45 | 益田街歩き | 安全管理・進行管理 | 街歩き・データ記録 |
| 1:45-2:00 | まとめ・振り返り | データ確認支援・総括 | データ確認・質疑応答 |

## 📋 各フェーズの詳細

### Phase 1: 開始・説明（0:00-0:05）

#### 司会者の行動指針
1. **開会挨拶**（1分）
   - 参加者への歓迎
   - 自己紹介とスタッフ紹介
   
2. **ハンズオン概要説明**（3分）
   - 今日の目的と流れ
   - 期待される成果
   - 安全に関する注意事項
   
3. **質問・サポート方法の説明**（1分）
   - スタッフへの相談方法
   - 緊急時の対応

#### チェックポイント
- [ ] 全参加者が着席している
- [ ] 配布資料が行き渡っている
- [ ] 注意事項が伝わっている

### Phase 2: GPS機器セットアップ（0:05-0:30）

#### 司会者の行動指針
1. **セットアップ手順の説明**（5分）
   - 機器の確認方法
   - 基本的な操作手順
   - 注意すべきポイント
   
2. **実作業の開始**（20分）
   - 参加者の進捗確認（5分間隔）
   - 個別サポートの実施
   - 遅れている参加者への重点支援

#### 時間調整のポイント
- **15分経過時点**: 70%以上の参加者が完了していることを確認
- **20分経過時点**: 遅れている参加者への集中サポート
- **25分経過時点**: 必要に応じて予備機器への交換検討

#### チェックポイント
- [ ] 全参加者が機器を正常に起動できている
- [ ] 基本設定が完了している
- [ ] 接続テストが成功している

### Phase 3: 動作確認（0:30-0:45）

#### 司会者の行動指針
1. **確認手順の説明**（3分）
   - GPS信号受信の確認方法
   - データ送信テストの手順
   
2. **実際の確認作業**（10分）
   - 参加者の作業支援
   - 問題発生時の対応
   
3. **結果の確認**（2分）
   - 全体の完了状況確認
   - 街歩き準備の最終チェック

#### チェックポイント
- [ ] GPS信号が正常に受信できている
- [ ] データ送信が成功している
- [ ] 全参加者が街歩き可能な状態

### Phase 4: 益田街歩き（0:45-1:45）

#### 司会者の行動指針
1. **出発前の説明**（5分）
   - 散策ルートの確認
   - 安全に関する注意事項
   - チェックポイントの説明
   
2. **街歩きの実施**（50分）
   - グループの先頭・最後尾での安全管理
   - 各チェックポイントでの確認
   - 遅れている参加者への配慮
   
3. **会場への帰着**（5分）
   - 参加者の人数確認
   - 機器の状態確認

#### 安全管理のポイント
- 交通量の多い場所での注意喚起
- 参加者の体調確認
- 機器の取扱い状況の確認

#### チェックポイント
- [ ] 全参加者が安全に移動している
- [ ] GPS機器が正常に動作している
- [ ] 予定ルートを適切に進行している

### Phase 5: まとめ・振り返り（1:45-2:00）

#### 司会者の行動指針
1. **データ確認の支援**（8分）
   - 取得データの確認方法説明
   - 個別の確認サポート
   
2. **全体の振り返り**（5分）
   - 今日の成果の確認
   - 学んだことの共有
   
3. **質疑応答・終了**（2分）
   - 質問への回答
   - 機材回収
   - 終了挨拶

#### チェックポイント
- [ ] 全参加者がデータを確認できている
- [ ] 機材が全て回収されている
- [ ] 参加者が満足している

## 🔄 時間調整の方法

### 進行が早い場合
- 各チェックポイントでの詳細説明を追加
- 参加者同士の情報共有時間を設ける
- 追加の質疑応答時間を設ける

### 進行が遅い場合
- 個別サポートを並行実施
- 予備機器への交換を積極的に実施
- 街歩きルートの短縮を検討
- まとめ時間の調整

## 📞 問題エスカレーションの基準

### 技術サポートへのエスカレーション
- 5分以上解決しない技術的問題
- 複数の参加者で同じ問題が発生
- 機器の故障が疑われる場合

### 全体進行の調整が必要な場合
- 30%以上の参加者が遅れている
- 安全上の問題が発生
- 天候等の外的要因による変更が必要
